# QA Phase 2：推薦卡、聊聊、地圖、收藏、診斷

適用建置：TestFlight QA 版（`ROAMIE_QA_BUILD=1` / `VITE_ROAMIE_QA=1`）。  
登入方式見 [QA_TEST_LOGIN.md](./QA_TEST_LOGIN.md)。

**記錄方式**：每項標 ✅ / ❌；失敗時附螢幕截圖 + 診斷 JSON（見第 5 節）。

---

## 通過標準（全頁共通）

| 檢查 | 通過 | 失敗（需回報） |
|------|------|----------------|
| 店名 | 可在 Google Maps 搜到的真實店名 | `附近散步點`、`附近咖啡`、`安靜咖啡` 等模板名 |
| 座標 | 點進詳情／導航有合理位置 | 無座標、座標在海上／錯城市 |
| 圖片 | 多張卡封面 **不全部相同**；優先為 Google Place Photo | 全部同一張 Unsplash／預設圖 |
| 營業 | 顯示營業中／休息／打烊等合理狀態 | 長期 `unknown` 或明顯錯誤 |
| 距離 | 與目前定位大致合理（步行／開車分鐘數可信） | 全部相同距離、或明顯離你很遠卻寫「附近」 |
| 資料來源 | 診斷中 `source_type` 為 `google_places`，`is_verified_real_place: true` | `mock` / `fallback` 且 `fallback_mock_triggered: true` |

程式會過濾假地名（`src/lib/place-verification.ts`）：名稱符合 `/附近.*散步|附近.*咖啡/` 者不應出現在 AI 推薦卡。

## 1. 首頁推薦卡

**前置**：已 QA 登入；系統允許定位（或確認使用 fallback 時距離僅供參考）。

| # | 步驟 | 預期 |
|---|------|------|
| 1.1 | 進首頁，等「附近推薦」載入完成 | 出現多張店卡，非長期空白 |
| 1.2 | 看 3 張以上店名 | 皆為真實店名（可選一兩個在 Google Maps 搜尋核對） |
| 1.3 | 比對封面圖 | 至少 2 張圖 **明顯不同**（非同一張重複） |
| 1.4 | 點開或看卡片上的營業／距離 | 營業狀態合理；距離與你所在區域相符 |
| 1.5 | （可選）點心情 **「一個人」** 或 **「下雨天」** | 導向推薦結果頁；卡片仍為真實地點，非假資料 |

**失敗時**：在任一推薦卡上展開 **診斷** →「複製診斷」或「下載 diagnostics.json」。

---

## 2. 聊聊頁

**前置**：首頁或聊聊已能取得定位（聊聊會帶入 context）。

| # | 步驟 | 操作 | 預期 |
|---|------|------|------|
| 2.1 | 進 **聊聊** | — | 可正常輸入與看到快捷 chip |
| 2.2 | 點 chip **「一個人」**（探索階段） | 送出後等 AI 回覆 | 回覆含 **地點卡片**；店名真實、有座標 |
| 2.3 | 點 **「下雨天可以去哪」**（預設 chip） | 同上 | 室內／雨天相關 **真實** 地點，非模板名 |
| 2.4 | 手動輸入並送出 | `我想找附近適合一個人待的咖啡廳` | AI 回覆咖啡廳等 **真實店** 卡片 |
| 2.5 | 負向檢查 | 掃描卡片標題 | **不得** 出現 `附近散步點`、`附近咖啡` 等 |

> 聊聊頁沒有單獨的「下雨天」chip；雨天請用 **「下雨天可以去哪」** 或首頁心情 **「下雨天」**。

**失敗時**：在 Roamie 回覆區的地點卡上 → 診斷面板 → 複製／下載 diagnostics.json。

## 3. 探索地圖

| # | 步驟 | 預期 |
|---|------|------|
| 3.1 | 進 **探索地圖**，允許定位 | 地圖載入、出現推薦列表／標記 |
| 3.2 | **停留 30 秒** 不操作 | 列表 **不會** 每隔幾秒整批閃爍重載 |
| 3.3 | **拖曳、縮放** 地圖 | 地圖流暢；不會無限迴圈刷新 |
| 3.4 | 看底部／側邊推薦卡 | 真實店名；圖片不全相同 |
| 3.5 | 點 1～2 張卡進詳情 | 地址、營業、距離合理 |

**失敗時**：地圖頁推薦卡區 →「下載 diagnostics.json」或單卡診斷。

## 4. 收藏頁

| # | 步驟 | 預期 |
|---|------|------|
| 4.1 | 進 **收藏**（無行程、無地點時） | **沒有**「建立空白行程」按鈕 |
| 4.2 | 行程分頁空狀態 | 僅 **「規劃第一趟旅程」**（→ 聊聊） |
| 4.3 | 同上 | **「去探索附近地點」**（→ 地圖） |
| 4.4 | 已有行程時 | 列表正常；無多餘「空白建立」入口 |

## 5. 診斷資料（卡片異常時必做）

QA TestFlight 建置（`ROAMIE_QA_BUILD=1` 或 `.env` 內 `VITE_ROAMIE_QA=1`）會開啟推薦卡診斷 UI。  
各頁頂部為 **琥珀色「QA 推薦診斷」區**（複製快照／下載 diagnostics.json）；每張卡下方另有細項（`data` / `image` / `place_id` 等）。

| 動作 | 位置 |
|------|------|
| **複製診斷** | 首頁推薦卡／聊聊回覆卡／地圖推薦卡 → 展開「診斷」 |
| **下載 diagnostics.json** | 同上區塊按鈕；或一次彙整多張卡 |

回報時請附上：

1. `build_version`、`environment`
2. 異常卡的 `title`、`place_id`、`source_type`、`is_verified_real_place`
3. `fallback_mock_triggered`、`fallback_reason`、`api_error_log`
4. `photo_source`、`opening_hours_source`、`distance_source`
5. 簡述操作路徑（例如：聊聊 → 一個人 → 第三張卡）

**登入頁 QA 診斷**（`step` / `hasSession`）僅用於測試登入問題，與推薦卡無關。

## 常見失敗原因（給開發對照）

| 現象 | 可能原因 |
|------|----------|
| 全部同一張圖 | Place Photo API 失敗，退回預設圖；看 `photo_fallback_reason` |
| 假店名 | 曾走 `curated_fallback` / mock；API 或 AI 未回傳可驗證地點 |
| 距離不對 | 定位拒絕或 fallback GPS；看 `location_source` |
| 地圖一直刷新 | 定位抖動或搜尋 debounce 異常（回報操作錄影） |
| 看不到診斷按鈕 | 非 QA 建置且未設 `VITE_DEBUG_DIAGNOSTICS=1` |
